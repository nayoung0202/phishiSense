# 프로젝트 생성 화면 개편 계획

## 목표
- 프로젝트 생성 페이지를 단일 화면 구성으로 재설계하고 요청 사양에 맞는 UI와 기능을 구현합니다.

## 주요 구현 항목
1. 좌측 폼: 기본정보, 대상 선택, 템플릿/도메인, 일정/운영, 안전/알림 섹션을 단일 페이지에 배치하고 필수 검증 추가.
2. 우측 미리보기: 실시간 카드/차트/대상 샘플/축약 대시보드 UI 구성.
3. 상단/하단 고정 액션 바: 임시 저장, 템플릿 미리보기, 테스트 메일, 취소, 생성/예약/바로 시작 버튼 로직 구현.
4. 미리보기 API 통신: `/api/projects/preview` 호출로 대상 변경 시 집계 갱신, 캐시 및 로딩 처리.
5. 테스트 메일 API `/api/projects/test-send` 연동 및 토스트 출력.
6. 생성/예약/상태 전이 API 호출 및 오류 처리(코드별 메시지, 빨간 바 요약).
7. 자동 저장, 일정 분기 배지, 템플릿/도메인 유효성, 중복 대상 경고 등 부가 기능.
8. 서버 측 엔드포인트 로직과 검증 추가.

## 작업 순서
1. 현행 프로젝트 생성 페이지 구조 분석 및 컴포넌트 분리 계획 수립.
2. 서버 API 스켈레톤 추가: preview, test-send, status 전이, 기존 create 보강.
3. 클라이언트 상태/서비스 레이어 구축: React Query mutation/query, 캐시 정책.
4. UI 구성: 폼, 미리보기 패널, 상/하단 액션 바, 차트/표.
5. 검증 및 에러 처리 흐름 구현: 필수값, 변수 매핑, 중복 대상.
6. 자동 저장/실시간 미리보기/토스트 등 상호작용 구현.
7. 통합 테스트(서버) 및 주요 컴포넌트 테스트 추가.
8. QA 체크리스트 작성 및 문서화.

## 산출물
- 코드 변경(클라이언트/서버)
- 필요 시 추가 문서(옵션)
