# ADR-0001: docs를 공식 기준으로 사용한다

- 상태: 승인
- 날짜: 2026-03-16

## 배경

기존 문서 구조에는 아키텍처 문서, 연동용 핸드오프 문서, 조사 로그, 작업 이력성 문서가 혼재되어 있었습니다. 이 상태에서는 어떤 문서가 공식 기준인지 즉시 판단하기 어렵고, 유지보수 비용이 계속 커집니다.

## 결정

- `docs/`를 저장소의 공식 기준 문서(SSOT)로 사용합니다.
- `plans/`는 로컬 작업 메모와 임시 판단을 기록하는 공간으로만 사용합니다.
- `outputs/`는 파일 산출물이 꼭 필요할 때만 선택적으로 사용합니다.
- 문서 구조는 `product`, `requirements`, `architecture`, `api`, `policy`, `ops`, `adr` 중심으로 단순화합니다.
- 일회성 핸드오프 문서와 로그성 문서는 공식 문서에 필요한 내용만 흡수하고 별도 보관하지 않습니다.
- README는 저장소 입구 문서로만 유지하고, 상세 기준은 `docs/README.md`에서 안내합니다.

## 결과

- 공식 기준 문서를 찾는 비용이 줄어듭니다.
- 작업 메모와 공식 기준이 섞이지 않습니다.
- 구조 변경 시 어떤 문서를 갱신해야 하는지 판단이 쉬워집니다.
- 상세 작업 이력은 Git 기록과 `plans/`로 분리합니다.
