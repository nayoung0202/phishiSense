# 프로젝트 관리 고도화 진행 현황

## 1단계 — 백엔드/데이터 정비
- `projects` 스키마에 `fiscalYear`, `fiscalQuarter`, `weekOfYear[]` 필드를 추가해 프로젝트 생성/수정 시 자동 계산되도록 구성했습니다.
- `/api/projects`, `/api/projects/quarter-stats`, `/api/projects/calendar`, `PATCH /api/projects/:id` 엔드포인트를 확장해 연도/분기/상태/검색 기반 조회와 일정 변경을 지원합니다.
- 시드 데이터를 2025년 분기 단위로 보강해 완료·진행중·예약 프로젝트가 모두 포함되도록 정리했습니다.

## 2단계 — 프런트 기본 뷰 개편
- 상단 필터 바에서 연도·분기·상태·검색어를 통합 관리하고 리스트/보드/캘린더 전환이 가능한 뼈대를 구축했습니다.
- 캘린더 월간/타임라인 초기 UI를 구성하고, 프로젝트 비교·보고서 미리보기·날짜 모달·상세 패널을 연결했습니다.

## 3단계 — 캘린더/타임라인 개선
- 월간·타임라인 뷰에 가상 스크롤을 적용해 많은 데이터가 있어도 렌더링 부담을 줄였습니다.
- 월간 캘린더를 단일 월 슬라이드 형태(`<` `>` 네비게이션)로 개편하고, 프로젝트를 단일 막대로 표현해 기간을 직관적으로 확인할 수 있습니다.
- 타임라인에서는 드래그로 주차 이동이 가능하며, 시작/종료 ±1일 조정 버튼과 상태별 배지 색상, 다양한 접근성 라벨/툴팁을 추가했습니다.

## 다음 단계 제안
- 분기 헤더 일괄 액션(예약 시작/중지, 템플릿 교체)과 사이드패널 편집 기능 확장.
- 캘린더/타임라인의 축약(`+n`) 뷰와 가상화에 키보드 탐색, 색각 보조를 포함한 접근성 보완.
