# SMTP 관리 UI 자동 테넌트 적용 계획

## 목표
- 테넌트 입력 UI 제거 후 자동 결정 로직을 도입하여 "현재 테넌트" 배지만 노출한다.
- SMTP 설정 로딩/저장/테스트 흐름을 자동 테넌트 ID 기반으로 유지하고 비밀번호/보안 요건을 준수한다.
- 백엔드 SSRF 방지 로직과 REST 라우트를 점검해 tenantId 흐름과 보안 요건이 충족되는지 확인한다.

## 작업 단계
1. **현황 파악 및 영향 범위 조사**: SMTP 페이지, API 호출 유틸, 관련 라우팅/컴포넌트를 파악하고 기존 테넌트 선택 UI/로직을 이해한다.
2. **프론트엔드 로직/UX 수정**: tenantId 자동 결정 헬퍼를 도입하고, 헤더 배지와 재조회 버튼, password 입력 초기화 등 상태 제어를 구현한다. 필요 시 컴포넌트/스타일을 정리한다.
3. **백엔드 라우트/보안 확인**: SSRF 방지 유틸과 SMTP 라우트를 확인해 tenantId 기반 동작, 비밀번호 비노출, 포트/도메인 검증 등이 충족되는지 점검하고 필요한 보완을 적용한다.
4. **검증 및 문서화**: 수동 점검(렌더링, 새로고침, 저장/테스트 흐름) 및 필요한 스크립트/테스트를 실행하고 결과를 기록한다.
