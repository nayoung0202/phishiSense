# 프로젝트 부서 표시 동기화 작업 계획

## 목표
프로젝트 생성 시 선택한 부서가 프로젝트 관리 화면(목록/상세 등)에서 동일하게 표시되도록 데이터 저장 및 표시 로직을 일치시킨다.

## 작업 범위
- 프로젝트 생성 폼에서 전송하는 부서 필드 확인
- 프로젝트 API 저장 스키마/매핑 확인
- 프로젝트 목록/상세 UI의 부서 표시 소스 확인
- 불일치 지점 수정 및 타입 정합성 점검
- 관련 테스트 또는 타입 체크로 회귀 확인

## 수행 단계
1. 생성 요청 페이로드와 서버 저장 로직의 부서 필드를 추적한다.
2. 프로젝트 조회 응답에서 부서 값이 누락/변형되는지 점검한다.
3. 프로젝트 상세/관리 UI가 동일한 필드를 참조하도록 수정한다.
4. 타입 체크 또는 관련 테스트를 실행해 변경 영향을 검증한다.
