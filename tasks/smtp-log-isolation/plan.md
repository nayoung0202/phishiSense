# SMTP 로그 분리 작업 계획

1. 현재 SMTP 관리 페이지와 등록 폼 구조, 관련 상태 관리 및 데이터 로딩 로직을 파악한다.
2. SMTP 로그가 공유되는 원인을 규명하고, 각 SMTP 엔트리별로 독립된 로그를 유지할 수 있는 데이터 구조 또는 상태 전환 방식을 설계한다.
3. 필요한 경우 서버 API 또는 클라이언트 저장소 로직을 수정해 SMTP별 고유 로그를 반환 혹은 필터링하도록 변경한다.
4. UI에서 해당 변경 사항을 반영하여 특정 SMTP 등록 화면에서 다른 SMTP의 로그가 노출되지 않도록 조정한다.
5. 변경 사항을 검증하고 필요한 테스트나 타입 검사를 수행한다.
