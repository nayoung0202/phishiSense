# 프로젝트 상세 부서별 분포 선택 부서 기준 전환

## 목표
부서별 분포 차트를 발송 데이터가 아닌 프로젝트 생성 시 선택한 부서(`departmentTags`) 기준으로 표시한다.

## 작업 범위
- ProjectDetail의 부서 분포 데이터 계산 로직 변경
- 차트 툴팁/라벨을 선택 부서 의미에 맞게 정리
- 빈 상태 메시지 정비
- 타입 체크로 안정성 검증

## 수행 단계
1. `departmentTags` 기반으로 차트 데이터를 구성한다.
2. 발송 수/대상자 수 의미를 제거하고 선택 기준 표시로 변경한다.
3. 선택 부서가 없을 때의 문구를 정리한다.
4. `npm run check`를 실행한다.
