# 템플릿 관리 페이지 메일 블록 개편 계획

1. 현행 템플릿 관리 관련 화면(`client/src/pages` 및 연계 컴포넌트)을 살펴보고 새 템플릿/수정 흐름이 어떻게 구성되어 있는지 파악한다.
2. 요구사항에 맞춰 "메일 본문"과 "메일" 두 블록을 구성할 수 있도록 UI 구조와 상태를 설계한다.
3. 가짜 악성 페이지 링크와 피싱 콘텐츠, 클릭 이후 노출되는 실제 악성 페이지 내용을 포함하는 더미 템플릿 데이터를 3건 정의한다.
4. 템플릿 생성/수정 폼과 목록 혹은 프리뷰에서 위 더미 데이터를 활용해 UX를 검증하고 필요한 스타일을 정리한다.
5. 변경된 기능을 점검하고 필요한 테스트나 문서를 갱신한다.
