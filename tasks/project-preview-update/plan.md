# 프로젝트 미리보기 엔드포인트 수정 계획

1. 현재 `/api/projects/preview` 관련 서버 라우트와 클라이언트 호출 위치를 조사해 구조와 데이터 흐름을 파악한다.
2. Express 라우트를 `/api/projects/:id/preview` 경로 및 POST JSON 본문을 수신하도록 바꾸고, 필요한 유효성 검증과 의존 모듈을 조정한다.
3. 클라이언트에서 해당 엔드포인트를 호출하는 로직을 POST JSON 호출로 수정하고 관련 타입을 업데이트한다.
4. 필요 시 테스트나 타입 검사를 보완하고 로컬 점검 명령(`npm run check` 등)을 검토한다.
