# SMTP 신규 등록 시 테넌트 ID 고유화 계획

1. SMTP 등록 페이지(SmtpCreatePage)에서 사용 중인 테넌트 ID 결정 로직을 파악하고, 현재 자동 테넌트 훅이 어떻게 동작하는지 확인한다.
2. 신규 등록 시 기존 테넌트를 덮어쓰지 않도록 고유한 tenantId를 생성/할당하는 전략을 설계한다. 필요하다면 tenant 헬퍼에 생성 함수를 추가한다.
3. 생성한 tenantId가 폼과 API 호출에 일관되게 전달되도록 컴포넌트를 수정하고, 정상적으로 새로운 레코드가 만들어지는지 확인한다.
4. 타입 검사를 실행하거나 관련 기능을 수동으로 검토해 회귀를 방지한다.
