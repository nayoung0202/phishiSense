# 프로젝트 생성 부서 태그 제거 및 대상 기반 부서 자동 산출 계획

## 목표
- 프로젝트 생성 화면에서 수동 `부서 태그` 선택 기능을 제거한다.
- 프로젝트의 부서 정보(`department`, `departmentTags`)를 선택된 대상(targetIds)의 부서로 자동 산출한다.
- 프로젝트 상세의 부서별 분포/관리 화면 부서 표시가 선택 대상 기준으로 동작하도록 보장한다.

## 작업 단계
1. 프론트 `ProjectCreate`에서 부서 태그 UI/검증/상태 제거
2. 서버 부서 산출 유틸 추가(대상 부서 문자열 -> 태그 배열)
3. `/api/projects` POST/PATCH에서 targetIds 기준 부서 자동 반영
4. 프로젝트 상세 분포 기준을 action log 대상 부서 기반으로 전환
5. 타입 검사/테스트 실행 및 커밋

## 확인 항목
- 프로젝트 생성 시 부서 태그 입력 없이 저장 가능한지
- 생성된 프로젝트의 부서 표시가 선택 대상 부서로 노출되는지
- 프로젝트 상세 `부서별 분포`가 선택 대상 기준으로 표시되는지
