# Azure VM 메모리 급증 원인 분석 계획

1. 서버/클라이언트 모두에서 메모리 사용량이 비정상적으로 증가할 수 있는 구조(무한 폴링, 대량 상태 보관, 메모리 누수 가능 API 등)를 코드 차원에서 탐색한다.
2. 백엔드 Express 서버의 주요 엔드포인트, 데이터 적재 방식(메모리 DB/캐시) 및 장시간 실행 시 누적되는 자료구조를 확인한다.
3. 프론트엔드 React/Vite 앱에서 비효율적인 데이터 페칭 루프나 거대 메모리 사용 UI가 있는지 조사한다.
4. 문제 지점을 요약하고, 대량 접속을 견딜 수 있도록 개선하거나 운영 레벨에서 취할 수 있는 조치(예: DB 이동, 캐시 전략)를 제안한다.
