# 전체 데이터 비휘발 DB 전환 계획

1. 현재 MemStorage로 관리 중인 엔터티(템플릿, 프로젝트, 타겟, 훈련 페이지 등)를 정리하고 Drizzle 스키마에 대응되는 테이블 정의를 검증한다.
2. 서버 측 데이터 접근 레이어를 MemStorage에서 Drizzle 기반 DAO로 전환하기 위한 구조(예: repository 모듈)를 설계하고, 우선 CRUD API가 많은 리소스부터 단계적으로 마이그레이션한다.
3. SMTP 관리 포함 모든 관련 API를 DB를 사용하도록 수정하고, `.env`/마이그레이션 안내를 업데이트한 뒤 타입 검사 및 기본 동작을 확인한다.
