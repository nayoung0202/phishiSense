# 템플릿 메일/악성 본문 연계 및 개인정보 유도 문구 추가 계획

1. 현재 `server/storage.ts`에 시드된 템플릿 3건의 메일 본문/악성 본문 내용을 파악해 링크 구조와 CTA 문구를 정리한다.
2. 메일 본문에 악성 페이지로 이어지는 링크 또는 버튼을 명확히 배치하고, 악성 본문에는 개인정보 입력 유도 문구 및 UI 요소를 강화하도록 수정안을 수립한다.
3. Drizzle 시드 데이터를 직접 수정해 각 템플릿의 `body`와 `maliciousPageContent`를 요구사항에 맞게 업데이트하고, 타입 검사가 통과하는지 확인한다.
