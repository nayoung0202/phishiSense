# 작업 계획

## 목표
- 테스트 메일 버튼 링크가 추적 URL로 치환되지 않도록 하고, 프로젝트 훈련 안내 링크(`/t/<token>`)로 이동하도록 보장한다.

## 작업 항목
1. 테스트 메일/프로젝트 발송에서 링크 치환 로직과 템플릿 처리 순서를 점검한다.
2. 추적 링크로 덮어쓰는 코드(추적용 링크 변환 등)를 찾아 훈련 안내 링크는 제외 처리한다.
3. 수정 후 테스트 메일 발송 응답/메일 본문에서 링크가 `/t/<token>`으로 유지되는지 확인한다.

## 검증 방법
- 테스트 메일 발송 후 본문 HTML을 확인해 `APP_BASE_URL/t/<token>`이 유지되는지 확인.
- 테스트 메일 발송 후 본문 HTML을 확인해 `APP_BASE_URL/t/<token>`이 유지되는지 확인.
- 수신 메일에서 버튼 클릭 시 `/t/<token>`으로 이동하는지 확인.
