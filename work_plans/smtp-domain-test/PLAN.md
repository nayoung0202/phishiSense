# 작업 계획

## 목표
- SMTP 관리페이지 테스트 발송 시 계정 아이디와 수신 이메일이 동일하지 않아도, 도메인(@뒤)이 같으면 발송되도록 변경한다.

## 작업 항목
1. SMTP 테스트 발송 요청 검증 로직을 찾아 동일 계정 제한 조건을 확인한다.
2. 조건을 "아이디와 수신자가 동일"에서 "도메인 일치"로 완화한다.
3. 실패 메시지를 도메인 기준 안내로 갱신한다.

## 검증 방법
- SMTP 계정: user@company.com
- 수신자: other@company.com → 통과
- 수신자: other@gmail.com → 차단
