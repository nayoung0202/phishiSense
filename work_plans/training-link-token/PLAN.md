# 작업 계획

## 목표
- 프로젝트별 훈련 안내 링크 토큰 생성 및 고정 매핑
- 공개 라우트 `/t/[token]`로 훈련 안내 HTML 렌더링
- 이메일 본문에 링크 치환/추가
- 스키마 반영 및 발송 흐름 연결

## 작업 순서
1. 현행 코드 구조 확인 (프로젝트/템플릿/발송 흐름, 스토리지/라우트)
2. 스키마에 `projects.training_link_token` 추가 및 고유 인덱스 정의
3. 프로젝트 생성/조회 시 토큰 생성 로직 추가
4. 공개 라우트 `/t/[token]` 구현 및 404/비활성 처리
5. 이메일 본문 링크 치환/append 로직 구현
6. 발송 API에서 프로젝트 기반 링크 주입 흐름 연결
7. 간단한 수동 검증 방법 정리 (curl 포함)
