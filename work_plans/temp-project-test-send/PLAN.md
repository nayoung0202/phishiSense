# 작업 계획

## 목표
- 프로젝트 생성 화면에서 테스트 메일 발송 시 임시 프로젝트를 먼저 저장하고, 해당 프로젝트 ID로 발송한다.
- 프로젝트 관리 화면에서 임시 프로젝트 상태가 "임시"로 표시되도록 한다.

## 작업 항목
1. 프로젝트 생성 화면에서 임시 저장/업데이트 후 테스트 발송하는 흐름을 추가한다.
2. 프로젝트 상태 "임시"를 프로젝트 목록/상세 화면에 표시하도록 상태 매핑을 확장한다.
3. 상태 필터/쿼리 매핑에 "임시"를 반영한다.

## 검증 방법
- 프로젝트 생성 화면에서 테스트 메일 발송 후 프로젝트 목록에 "임시" 상태로 표시되는지 확인.
- 테스트 메일 본문 링크가 `/t/<token>`으로 생성되는지 확인.
