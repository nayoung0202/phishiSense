# 작업 계획

## 목표
- 악성메일 본문 → 훈련안내 페이지로 이동한 뒤 재접속 시에도 메일 링크(악성 본문)부터 다시 시작하도록 한다.

## 작업 항목
1. 악성메일 본문 라우트에서 플로우 쿠키를 발급한다.
2. 훈련안내 라우트에서 플로우 쿠키 검증 후 없으면 악성메일 본문으로 리다이렉트한다.
3. 훈련안내 렌더링 이후 플로우 쿠키를 제거해 재접속 시 다시 시작하게 한다.

## 검증 방법
- `/p/<token>` 방문 시 쿠키가 설정되는지 확인.
- `/t/<token>` 직접 접근 시 `/p/<token>`로 리다이렉트되는지 확인.
- `/p/<token>` → `/t/<token>` 이동 후 새로고침 시 `/p/<token>`로 돌아가는지 확인.
