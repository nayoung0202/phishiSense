# 작업 계획

## 목표
- 메일 본문 링크 클릭 시 악성메일 본문(템플릿 maliciousPageContent)을 먼저 렌더링한다.
- 악성메일 본문에서 훈련 안내 페이지(`/t/<token>`)로 이동할 수 있도록 링크를 주입한다.

## 작업 항목
1. 링크 생성/주입 유틸을 보강해 악성 페이지용 링크와 이메일용 링크를 분리한다.
2. 악성메일 본문을 렌더링하는 공개 라우트(`/p/[token]`)를 추가한다.
3. 테스트 메일 발송에서 이메일 링크가 `/p/<token>`로 생성되도록 변경한다.

## 검증 방법
- 테스트 메일 발송 후 버튼 링크가 `/p/<token>`인지 확인.
- `/p/<token>` 접속 시 악성메일 본문 HTML이 렌더링되는지 확인.
- 악성메일 본문 하단(또는 플레이스홀더)에 훈련 안내 링크(`/t/<token>`)가 노출되는지 확인.
- `/t/<token>` 접속 시 훈련 안내 HTML이 렌더링되는지 확인.
