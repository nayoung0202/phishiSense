# 보고서 설정 선택형 생성 모달 통합 및 승인자 반영 계획

## 목표
- 프로젝트 목록/상세에서 동일한 보고서 생성 모달을 사용한다.
- 보고서 생성 시 보고서 설정 선택(`reportSettingId`)을 필수화한다.
- 선택한 보고서 설정의 승인자 정보가 실제 보고서 템플릿 데이터에 반영되도록 연동한다.

## 작업 범위
1. `src/components/ReportGenerateDialog.tsx` 공용 모달 컴포넌트 추가
2. `src/features/Projects.tsx`의 기존 보고서 모달 로직을 공용 컴포넌트로 교체
3. `src/features/ProjectDetail.tsx` 보고서 생성 버튼을 공용 모달 기반으로 교체
4. `src/app/api/reports/generate/route.ts` 요청 바디에 `reportSettingId` 필수 추가
5. `src/server/services/reportGenerator.ts`에서 보고서 설정 조회/로고/승인자 반영
6. `src/server/dao/reportInstanceDao.ts`의 `reportSettingId` 저장/업데이트 누락 보완
7. 관련 테스트(`route.test`, `reportGenerator.test`) 정합화

## 검증 기준
- 보고서 설정이 없으면 모달에서 생성이 차단되고 안내 문구가 보인다.
- 보고서 설정이 있으면 select에서 선택 후 생성 가능하다.
- Projects/ProjectDetail 모두 동일 모달 UX로 보고서 생성이 동작한다.
- 보고서 생성 API에 `reportSettingId`가 포함된다.
- 생성된 보고서 이력에 `reportSettingId`가 저장된다.
