# 작업 계획

## 목표
- 프로젝트 보고서 생성 기능(템플릿 기반 DOCX)을 서버 내부 모듈로 구현하고 프로젝트 관리 화면에서 다운로드할 수 있게 한다.

## 범위
- DB 스키마: report_templates, report_instances
- 서버 모듈: 보고서 생성기(입력: projectId, 출력: 파일 버퍼/다운로드 URL)
- API: 보고서 생성/다운로드, 템플릿 업로드(최소)
- 프런트: 프로젝트 관리/상세의 보고서 버튼 연결

## 작업 단계
1. 스키마/DAO/스토리지에 report 템플릿 및 인스턴스 모델을 추가한다.
2. 템플릿 파일 저장 경로와 다운로드 경로를 위한 파일 스토리지를 만든다.
3. Python 기반 DOCX 렌더러 스크립트를 추가하고 서버 모듈에서 호출한다.
4. 보고서 생성/다운로드 API를 구현한다.
5. 프로젝트 관리(및 상세)에서 보고서 생성 버튼을 API 호출로 연결한다.

## 검증
- 프로젝트 관리 화면에서 보고서 생성 클릭 시 DOCX 다운로드가 동작한다.
- report_instances에 생성 이력이 기록된다.
