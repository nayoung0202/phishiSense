# 보고서 템플릿/캡처 업로드 작업 계획

## 목표
- 보고서 생성 시 프로젝트별 캡처 4장을 필수로 업로드하고, 미설정/누락 시 생성 차단
- 회사명/로고는 환경변수로 설정하고 미설정 시 생성 차단
- 파일명 규칙: 회사명_프로젝트명_보고서_YYYYMMDD
- 템플릿은 docxtpl 기반 변수 치환과 도넛 차트 2종 유지

## 작업 단계
1. 전역 설정(환경변수) 적용: 회사명/로고 경로 검증 및 미설정 처리
2. 프로젝트 보고서 다이얼로그에 캡처 4장 업로드 UI 추가
3. 서버 저장/조회 및 보고서 생성 로직에 캡처·로고 이미지 삽입 처리
4. 템플릿 키 매핑 정리 및 샘플 템플릿 교체/적용
5. 검증/에러 메시지/다운로드 파일명 규칙 반영

## 체크 포인트
- 캡처 이미지 포맷/용량 제한 및 4장 필수 검증
- 로고/회사명 미설정 시 보고서 생성 차단
- 이미지 크기: 캡처 가로 16.5cm, 로고 6.54cm x 2cm
- 파일명: 회사명_프로젝트명_보고서_YYYYMMDD
