# 보고서 관리 메뉴 노출 및 기능 설명 정리 계획

## 요약
- 현재 `보고서 관리` 기능은 구현되어 있으나 사이드바에 메뉴가 없어 진입이 어려운 상태입니다.
- 사이드바에 `보고서 관리` 메뉴를 추가하고, 현재 구현된 보고서 설정/생성 흐름을 코드 기준으로 설명합니다.

## 구현 범위
1. `plans` 디렉터리 내 작업 계획 문서 생성
2. 사이드바 메뉴에 `보고서 관리`(` /reports/settings`) 항목 추가
3. 하위 경로까지 활성화 상태가 유지되도록 메뉴 활성화 로직 개선
4. 타입 체크 수행(`npm run check`)
5. 변경사항 커밋

## 검증 항목
- 사이드바에서 `보고서 관리` 메뉴가 보인다.
- `보고서 관리` 클릭 시 `/reports/settings`로 이동한다.
- `/reports/settings` 페이지 진입 시 목록 API 호출이 정상이다.
- 기존 메뉴 동작 회귀가 없다.
- `npm run check`가 통과한다.

## 비고
- 이번 작업은 UI 노출과 설명 제공에 집중하며, 보고서 생성 파이프라인의 데이터 연동 구조 변경은 포함하지 않습니다.
