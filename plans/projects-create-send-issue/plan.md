# 프로젝트 생성 직후 발송 실패 이슈 작업 계획

## 배경
- 증상: 프로젝트를 바로 생성한 뒤 발송이 동작하지 않음
- 비교군: 테스트 발송은 정상 동작

## 목표
- 프로젝트 생성 직후 발송 실패 원인을 코드/데이터 흐름 기준으로 식별
- 기존 테스트 발송 경로에 영향 없이 실제 발송 경로를 정상화
- 회귀를 방지할 최소 테스트 또는 검증 절차 추가

## 작업 단계
1. 프로젝트 생성 API/상태 저장 로직과 발송 트리거 로직의 연결 지점을 확인한다.
2. 테스트 발송과 실제 발송 경로의 입력값/검증/식별자 사용 차이를 비교한다.
3. 실패 원인을 재현 가능한 수준으로 좁히고 서버/클라이언트 중 필요한 코드를 수정한다.
4. 타입 검사 및 관련 테스트를 실행해 부작용을 확인한다.
5. 변경 내용을 정리하고 커밋한다.

## 확인 기준
- 프로젝트 생성 직후에도 실제 발송 API가 정상 응답을 반환한다.
- 테스트 발송 기능은 기존처럼 동작한다.
- `npm run check` 또는 관련 테스트가 통과한다.
