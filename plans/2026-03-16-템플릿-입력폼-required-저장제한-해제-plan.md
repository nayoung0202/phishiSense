# 템플릿 입력 폼 required 저장 제한 해제 계획

## 작업 목적
- 템플릿 생성 및 AI 생성 템플릿 적용 시 HTML 내부 입력 폼의 `required` 속성 때문에 저장이 막히는 문제를 해소한다.
- 템플릿 편집 단계에서는 입력값 미기재 상태로도 저장할 수 있게 하되, 실제 미리보기/훈련 링크 구조는 유지한다.

## 확인 범위
- `src/features/TemplateEdit.tsx`
- `src/components/ui/rich-text-editor.tsx`
- `src/components/template-preview-frame.tsx`
- 템플릿 저장 전 유효성 검사 또는 브라우저 기본 폼 검증이 발생하는 경로

## 작업 계획
- 저장 버튼 클릭 시 브라우저의 기본 폼 검증이 어떤 DOM에서 트리거되는지 확인한다.
- 템플릿 미리보기/에디터 내부 폼이 저장 UI를 가로채지 않도록 상호작용 또는 검증 속성을 조정한다.
- 관련 회귀 테스트를 추가해 템플릿 저장이 입력값 없이 가능한 상태를 보장한다.

## 검증 계획
- 악성메일본문에 `required` 입력이 있어도 저장 버튼이 정상 동작하는지 확인한다.
- 미리보기와 편집 모드의 일반 렌더링이 깨지지 않는지 함께 점검한다.
