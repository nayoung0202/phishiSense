# 작업 계획: 템플릿 필수 링크 토큰 검증 전환

## 목표
- UI 요소 기반 검증을 토큰 기반 검증으로 전환하고 저장/발송 차단, 상태 표시, 중복 삽입 제한을 반영한다.

## 범위
- 템플릿 편집 화면의 저장 전 검증 로직
- 템플릿 관리 목록의 상태 표시
- 필수 링크 블럭 중복 삽입 제한 및 블럭 편집 허용
- 자동 삽입 토글 제거

## 접근 방법
1. 현행 템플릿/에디터 관련 코드 위치 파악
2. 필수 토큰 규칙 정의 및 저장/발송 검증 흐름 수정
3. 상태 표시 문구 및 UI 업데이트, 토글 제거
4. 중복 삽입 방지 로직 추가 및 테스트 보완

## 산출물
- 관련 UI/로직 변경 코드
- 필요 시 테스트 및 목킹 업데이트

## 확인 항목
- 토큰 누락 시 저장/발송 차단 여부
- 포함됨/누락 상태 표시 정확성
- 중복 삽입 제한 동작
