# 악성메일본문 미리보기 배경 레이어 점검 계획

## 작업 목적
- 템플릿 관리에서 악성메일본문 미리보기 모달을 열었을 때 회색 또는 흰색 배경 레이어가 의도치 않게 보이는 원인을 확인한다.
- 원인이 UI 렌더링 구조나 미리보기 HTML 정규화 로직에 있다면 사용자가 자연스럽게 미리보기를 볼 수 있도록 수정한다.

## 확인 범위
- `src/components/TemplateCard.tsx`
- `src/components/template-preview-frame.tsx`
- `src/lib/templatePreview.ts`
- 템플릿 관리 화면에서 미리보기 모달과 샌드박스 프레임에 연결된 스타일 경로

## 작업 계획
- 템플릿 관리 화면에서 악성메일본문 미리보기 진입 경로와 렌더링 컴포넌트를 확인한다.
- 회색/흰색 레이어가 프레임 배경, 샌드박스 스타일, HTML 중첩 래퍼, 모달 오버레이 중 어디서 생기는지 분석한다.
- 원인이 확인되면 관련 컴포넌트 또는 정규화 로직을 수정하고 회귀 위험이 있는 테스트를 보강한다.

## 검증 계획
- 악성메일본문 미리보기에서 배경이 한 겹으로만 보이는지 관련 렌더링 테스트로 확인한다.
- 기존 메일 본문 미리보기와 AI 후보 미리보기 동작이 깨지지 않는지 함께 점검한다.
